API REST

API : APlication Programming Interface es un cojunto de rutinas que nos provee acceso a ciertas funcionalidades de un software en especifico

API NATIVAS: lo que viene integrado en el navegador, documento

API EXTERNAS: como por ejemplo los datos de formatos json en la pagina de github, facebook,etc

REST: Representational State Transfer o en espa√±ol Transferencia del estado representacional

es un conjunto de restricciones con las que podemos crear un estilo de arquitectura de software, migracion de MVC a una arquitectura rest

Se basa en las reglas del protocolo HTTP

CLIENTE Y SERVIDOR: estan separados o desacoplados, al cliente no le importa como el servidor procesa los datos y los devuelve en formato JSON, y los servidores no le interesa como el cliente va utilizar la informacion

SIN ESTADO: cada peticion recibida por el servidor se va a manejar de manera independiente lo que evita que tenga sessiones y hacer varias peticiones al mism tiempo

Cacheable: Una base de datos cache para evitar repetitivas conexiones hacia un mismo recurso del servidor y esto va impactar en el rendimiento de las aplicaciones

INTERFAZ UNIFORME: una interfaz generica ue permita la administracion entre la interaccion entre el cliente y el servidor de una manera uniforme significa que para cada peticion va a tener una URL unica y con las caracteristicas de como vamos a enviar la informacion, coocidos como los endpoints

VARIAS CAPAS: Desarrollo escalable, monitorizar el rendimiento y capas de seguridad
