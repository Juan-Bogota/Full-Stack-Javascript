Ajax no es una tecnologia en si mismo. E realidad, se trata de varias tecnologias que se unen:

  • HTML Y CSS para crear ua presentación basada en estándares
  • DOM para la interaccion y manipulacion dinamica de la presentación
  • HTML, XML y JSON para el intercambio asincrono de informacion
  • XMLHttpRequest o Fecth para el intercambio asincrono de informacion
  • JavaScriptPara unir todas las tecnologias

Es que AJax es asíncrono Qué significa esto. Normalmente cuando hacemos una petición y esperamos una respuesta en el navegador se queda cargando y hasta que no hemos recibido todos los datos pues no carga la página.

Hasta que no ha recibido todos los datos pues YouTube no se ve ni se puede utilizar entonces con Ayax lo que conseguimos es que se cargue la página y todos los datos que hagamos como petición al servidor lleguen más tarde porque como normalmente cuando hacemos una petición Ayax estamos haciendo una consulta a una base de datos o estamos haciendo una consulta a una API que nos devuelve una serie de datos.
Pues si estamos pidiendo muchos datos es posible que la petición tarde y puede tardar milisegundos o se pueda retrasar uno o dos segundos.
Entonces para qué durante ese tiempo la página siga funcionando. Lo que hace Ajax es trabajar de forma asíncrona y eso significa que no está sincronizado con la página
